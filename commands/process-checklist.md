# Command: process-checklist

Print the quality gate for a completed screen.

## Quality gate

- 01_canvas_system.md and 02_ui_tokens.md exist and are referenced
- Structure is clear and complete
- Wireframe is layout-only
- Canvas applied without layout drift
- UI mock is token-constrained
- Final mock is implementable without interpretation
- No system redefinition in screen files
- Change requests are classified and resolved upstream
